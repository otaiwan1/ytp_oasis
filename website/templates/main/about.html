About OASIS

Learn about the technology and research behind the system.

Project Overview

OASIS (Orthodontic AI Scan Image Search) is a research project developed under the YTP 2025 (Young Turing Program). The goal is to assist orthodontists by providing an AI-powered tool that can search for similar dental cases from a database of 3D scans and intraoral photos.

When a dentist encounters a new patient, they can upload the patient's 3D dental scan (STL file) and our system will find the most similar historical cases across 6 different AI models. The system supports both 3D point cloud analysis (SimCLR, Point-MAE) and 2D vision transformer approaches (DINOv2, DINOv3), including variants that use real iTero gallery photos for richer feature extraction.

Models

SimCLR (512-dim)

Self-supervised contrastive learning with EdgeConv (DGCNN) on 3D point clouds.

Point-MAE (384-dim)

Masked autoencoder on 3D point clouds with surface normals for self-supervised pre-training.

DINOv2 & DINOv3 (1024-dim)

Meta's Vision Transformers (ViT-L) applied to 5-angle rendered views of 3D meshes. CLS token pooling.

DINOv3 Gallery (2048-dim)

Uses real iTero intraoral photos with CLS + mean(patch) hybrid pooling for richer features.

DINOv3 Concat (10240-dim)

Concatenates all 5 views' hybrid embeddings to preserve per-view details (5 × 2048).

Data Pipeline

1
iTero Data Collection

~1300 iTero reports collected, filtered to 642 first-scan cases with 3D STL meshes and gallery photos.

2
3D Preprocessing

STL → point clouds (4096 pts, FPS sampling) for SimCLR/MAE, or 5-angle rendered images for DINOv2/v3.

3
Gallery Photo Extraction

5 real intraoral penta photos (1920×1080) extracted from iTero zip files for DINOv3 Gallery/Concat models.

4
Batch Embedding

Unified embed/ module generates embeddings for all 642 scans per model, saved as .npy + .json.

5
Validation & Search

Cosine similarity search with built-in validation system for measuring top-K retrieval accuracy per model.
